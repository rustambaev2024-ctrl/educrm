## 8 столпов UI — весь интерфейс строится по ним
Активные скилы: `frontend-design`, `design-system`, `shadcn`. Стек: TanStack Start + Tailwind v4 + shadcn/ui (new-york, radix, lucide). Всегда используй семантические токены (`bg-primary`, `text-muted-foreground`), `gap-*` вместо `space-y-*`, `size-*` вместо `w-* h-*`.

1. **Микроанимации и переходы.** Каждое действие имеет плавный отклик 150–300ms: карточки fade-in, панели выезжают, кнопка при нажатии «проваливается». Анимации shadcn Dialog/Sheet использовать как есть. Уважать `prefers-reduced-motion`.
2. **Типографика с иерархией.** С первого взгляда ясно: главное / второстепенное / подсказка. Разные размеры и веса (light/regular/semibold/bold), правильный line-height. Не делать всё одним размером.
3. **Состояния каждого элемента.** Минимум 5 состояний: обычное, hover, active, disabled, loading. Пользователь всегда понимает, можно ли кликнуть и что происходит.
4. **Пустые состояния.** Пустой список = компонент `Empty`: иконка + понятный текст («У вас пока нет студентов») + кнопка действия. Никогда просто белый экран.
5. **Обратная связь на каждое действие.** Клик → спиннер/«Сохраняем…» → `sonner` toast: зелёный «Сохранено» или красный с конкретной причиной. Ошибки прокидывают детали через `apiErrorMessage`, не голое «Xatolik».
6. **Детали для внимательного взгляда.** Аватар с инициалами — цвет детерминирован по имени, иконки выровнены с текстом (`data-icon`), чередование строк таблицы, тонкая цветная черта слева у активного пункта меню.
7. **Осмысленные цвета, не декоративные.** Красный = только опасность/удаление, зелёный = успех/активность, жёлтый = предупреждение, серый = неактивное. Цвет несёт информацию. Никаких `bg-blue-500` «потому что синий».
8. **Отзывчивость под контекст.** Не сжатый десктоп: на мобиле таблица → карточки, боковое меню → нижняя навигация, тач-таргеты ≥44px, модалки открываются снизу как `Drawer`/`Sheet`. student/ и parent/ порталы — mobile-first (`MobileLayout`).

### Подтверждения
Опасные и необратимые действия (удаление, отмена, возврат) подтверждаются через `ConfirmDialog` (`src/components/ui/confirm-dialog.tsx`) — НИКОГДА `window.confirm`.
